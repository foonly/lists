# App UX tweaks

## Done items

Done items should not be in their own section, they should stay in the list where they were, but be marked as done. This way we can keep the context of how they relate to other items.

There should be a "delete all done items" button at the bottom of the list, so you can easily clean up completed items without having to delete them one by one. It should use an icon to keep it small.

## Home view

The list display in the home view should show how many incomplete items of how many total there are in each list, so you can see at a glance which lists need attention.

The current time display is very chaotic, it seems to display quite random durations, not sure what's wrong, but it should display the duration since the last edit in the list.

## List view

There should be some padding before the first item in the list. It now displays a bit too close to the top bar.

The drag handle is not centered vertically.

## Add and edit item

The bottom widget for adding and editing items should be removed. We should instead have a "+" icon that adds items inline. Each group should have it's own "+" button, and there should be one for the ungrouped items. When you click it, a new item is added with an empty name and immediately goes into edit mode.

Editing should be possible inline as well for the main text.

Advanced edits for the text, quantity, unit and group should be done via an edit icon that opens a modal with all the edit fields. This way we can keep the main UI clean and simple, and still allow for more complex edits when needed.
